# Публикатор

Замыкает текстовый завод (Фаза 1): берёт готовый пост (текст + обложка-ФАЙЛ) и ставит его в
нативные «Отложенные» канала на слот контент-плана — аккаунтом-владельцем (userbot), как руками.
Telegram отправляет по расписанию; владелец проверяет/правит/одобряет в «Отложенных» до отправки.

> **Публикатор — это НЕ отдельный бот.** Это функция `/schedule` внутри Криейтора + коннектор
> `connectors/telegram_publish`. Отдельный бот был, но удалён за ненадобностью (владелец ничего не
> пересылает — Криейтор сам зовёт публикацию). Координацию всей цепи позже возьмёт оркестратор.

## КЛЮЧЕВОЕ РЕШЕНИЕ — публикуем ФАЙЛОМ (не URL/превью)
ГПТ рисует обложку как ФАЙЛ, Криейтор имеет файл + текст, Публикатор шлёт их userbot'ом:
- `send_file(фото, caption=текст)` — **фото + текст ОДНИМ сообщением**;
- если подпись длиннее лимита Telegram — **фото + текст ДВУМЯ сообщениями** (Telegram решает сам,
  без пред-проверки лимита).

**Никакого хостинга/URL/превью** (telegra.ph/0x0/catbox/imgbb, link-preview, invert_media) — это было
выкинуто (коммит 7d465b6): бесплатные хостинги нестабильны и ломали рабочее. «Обложка над текстом по
URL» — тупик, от него отказались. Рендер жирного/кастом-эмодзи — `tg_format.to_telegram_html(
custom_emoji=True)` + `parse_mode='html'` (премиум-аккаунт шлёт кастом-эмодзи).

## Как и чем постит
Аккаунт-публикатор **@Amanbabai228 / ЕвгенийП** (userbot, MTProto) — права владельца, лимитов Bot API
нет. Сессия `data/evgeniyp.session` (та же, что у аналитики: один аккаунт — read метрики + write пост).
Коннектор `connectors/telegram_publish/publish.py`: `publish()` (постановка), `notify()` (пинг на мейн),
`check()`, `scheduled_times()`. Приватный канал адресуется **инвайт-ссылкой** (PUBLISH_CHANNEL=
`https://t.me/+…`) — по имени НЕЛЬЗЯ (резолвится в чужой публичный канал).

## Контент-план (КОГДА)
Зеркало memory/post_standard §«Ритм недели» в `core/content_plan.py`: **Вт + Чт** — флагман (17:00),
**Пн/Ср/Пт** — короткий (15:00), **Сб/Вс** — нет. Формат — по длине поста; время/пояс — в `.env`
(`PUBLISH_TZ`, `PUBLISH_*_TIME`). Берёт ближайший слот, пропуская дни с уже стоящей отложкой.

## Безопасность (PLAN §6)
- **Детерминированно, без LLM** в цепочке публикации: `/schedule` зовёт код напрямую (agent_runtime
  `command_actions` → `creator_tools.publish_now`), модель не участвует, кредиты Claude не тратятся.
- **Не публикует вживую** — ставит в «Отложенные» + пишет владельцу на мейн (`PUBLISH_NOTIFY`).
  Финальный контроль — у владельца в нативных «Отложенных» (правит/отменяет/сдвигает).

## Поток
Криейтор `/post` (текст + обложка-файл через make_image, сохраняет драфт) → владелец смотрит →
Криейтор `/schedule` → бёрнер ставит файл фото+текст в отложку на слот + пишет на @Kanaki_K.
Всю цепь (Скаут→Криейтор→отложка) одной командой — `python run_pipeline.py` (`--skip-scout` — взять
последний бриф).

## Эксплуатация (.env)
`PUBLISH_CHANNEL` (инвайт-ссылка тест-канала), `TELEGRAM_SESSION`/`data/evgeniyp.session`,
`PUBLISH_NOTIFY` (@мейн), `PUBLISH_TZ=Europe/Berlin`, `PUBLISH_FLAGSHIP_TIME`/`PUBLISH_SHORT_TIME`,
`GPT_IMAGE_STUB=` (пусто = реальная генерация обложки через ГПТ-бёрнер). Аккаунт-публикатор должен быть
**админом** канала с правом постить (Premium — желателен).

## Грабли
- `/schedule` НЕ генерит обложку (только `/post`); берёт последнюю (`data/creator_last_cover.txt`).
- `make_image` (playwright) на Windows: запускать в отдельном потоке (run_pipeline так и делает).
- Приватный канал — только инвайт-ссылка/id, не имя.

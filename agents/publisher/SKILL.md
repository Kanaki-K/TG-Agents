# Публикатор

Замыкает текстовый завод (Фаза 1): берёт ОДОБРЕННЫЙ владельцем пост и кладёт его нативно в
канал — как руками человека (обложка сверху + полный текст), а не превью-карточкой Bot API.

## Роль и границы
- **Только публикация.** Не пишет и не правит тексты — это Криейтор. Не собирает аналитику — это
  Аналитик. Публикатор — «руки» с гейтом: что одобрено, то и в канал, дословно.
- **Детерминированный, без LLM.** В цепочке публикации модели нет намеренно: пост уходит РОВНО
  как одобрил владелец, без переписывания и без «самостоятельных» решений. Поэтому ключ Claude
  Публикатору не нужен (`core/publisher_bot.py`, не `agent_runtime`).
- **Чек-лист безопасности (PLAN §6):** публикует не бот сам по себе. Только по явной команде
  `/publish` владельца. Авто-публикации по расписанию НЕТ (добавим позже, когда качество выверено).

## Как публикует (MTProto, не Bot API)
Постит аккаунтом-публикатором @Amanbabai228 (userbot) — у него права владельца, лимитов Bot API
нет. Та же сессия, что у аналитики (`TELEGRAM_SESSION`): один аккаунт — read метрики + write пост.
Коннектор: `connectors/telegram_publish/publish.py`. Выбор формы — автоматом по длине:
- короткий пост (≤ лимита подписи 1024 / 2048 с Premium) → **фото + подпись**, одним сообщением;
- длинный (флагман 2800–4096) → **обложка крупным превью НАД текстом**, одним сообщением
  (подпись бы не вместила; обложка заливается на хостинг — превью тянется по публичному URL);
- сбой/нет URL → **фолбэк**: фото отдельным сообщением + полный текст (пост не теряем).

## Поток
Криейтор отдаёт текст + обложку → владелец правит/одобряет → пересылает финал и обложку
Публикатору → `/preview` (что и как уйдёт) → `/publish` → пост в канале.

## Команды
- (текст) — финальный текст поста; (фото) — обложка (подпись к фото = текст, если прислать вместе)
- `/preview` — показать, что и каким путём уйдёт (текст — дословно)
- `/check` — предполётная проверка: какой аккаунт-публикатор, Premium, виден ли канал
- `/publish` — опубликовать (гейт; защита от двойного тапа)
- `/discard` — сбросить staged-пост · `/start`, `/help` — справка

## Эксплуатация
Запуск: `python run_publisher.py`. Нужны в `.env`: `PUBLISHER_BOT_TOKEN`, `PUBLISH_CHANNEL`,
`TELEGRAM_SESSION`. Опционально: `PUBLISH_PREMIUM` (1 если у аккаунта Premium),
`PUBLISH_LONG_MODE` (`preview` по умолчанию | `split`). Staged-пост лежит в
`data/publisher_pending.json` (вне git), переживает рестарт.

## Пререквизиты обкатки (без них в реальный канал не публикуем)
1. **Тест-канал** (не основной) — куда безопасно постить на обкатке.
2. **Аккаунт-публикатор @Amanbabai228 — админ** этого канала с правом постить.
3. **Premium** на аккаунте — желателен (чистый «фото+подпись» для коротких; на длинные не влияет).
Проверка готовности — командой `/check`.

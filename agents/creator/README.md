# Агент: Криейтор

Пишет **готовые посты** по направлению от владельца (часто — по выводу Скаута/дип-ресёрча).
Продукт — драфт в голосе автора на правку. Замыкает «ресёрч → пост».

## Запуск
```
python run_creator.py
```
Команды: `/post` — пост по свежему брифу, формат под тему (или скажи, по какому); `/light` — лёгкий
философский (200–400 слов); `/formats` — предложить форматы под тему по данным; `/polish` — прислать
~готовый текст, оценит /10 и доведёт до 10/10; `/feedback` — пришли финал, сравнит со своим драфтом
и усвоит урок; `/derive` — вывести/обновить стандарт; `/apply_standard` — применить. Можно и просто прислать тему.

Выдача: пост приходит ОТДЕЛЬНЫМ сообщением, мета (формат/оценка/чего не хватает) — следом отдельно
(через маркер `[[SPLIT]]`), чтобы пост был чистым для копирования. Флагман целится в одно сообщение Telegram (2800–4096 знаков, идеал 3000–3800).

## Роль (важно)
Криейтор — **редактор + фактчекер, не со-автор**. Лучший пост = разворот ТЕЗИСОВ владельца в структуру
+ фактчек, а не сочинение с нуля по теме. Самокритику (актуальность/логика/польза + рубрика 7 критериев,
`content_manual.md` §10) он ведёт **ВНУТРИ и сразу выдаёт лучший результат** — без диалога про баллы и
без «как тебе?». Владелец не грейдит его в чате; Криейтор учится на правках (`/feedback`).

## Конфиг
- Модель: `claude-opus-4-8` (качество письма критично).
- `thinking: adaptive` — острее композиция и соблюдение красных линий (убери строку, чтобы выключить).
- `web_search: true` — фактчек цифр/дат/цен перед выдачей (самая дорогая ошибка — непроверенная цифра).
- Бот-токен: `CREATOR_BOT_TOKEN`. Ключ: `CREATOR_ANTHROPIC_KEY` (или общий).

## Инструменты (`core/creator_tools.py`)
Узкие, под письмо:
- `read_brief` / `list_briefs` — забирают бриф Скаута напрямую из `memory/briefs/` (стык «ресёрч → пост»).
- `find_posts` / `by_theme` — сверка с историей канала (анти-повтор, отсылки).
- `save_draft` / `read_draft` / `list_drafts` — архив драфтов в `memory/drafts/` (+ поднять свой под сравнение).
- `record_lesson` — усвоить урок из правки владельца в `memory/post_lessons.md`.
- `top_posts` / `by_dimension` / `themes_overview` — **сверка с данными Аналитика** (что заходит: ER по формату/времени/теме).
- `propose_standard` / `apply_standard` — вывод/обновление стандарта через гейт (см. ниже).
В контекст подаются: **`memory/content_manual.md` — ГЛАВНЫЙ мануал** (голос «Вы», типографика,
флагман как эссе-убеждение от антитезы, хук-сцена, контр-блок, запретник AI, размер 2800–4096 знаков (одно сообщение Telegram), футер; приоритет при конфликтах),
`memory/brand.md` (аудитория/красные линии), `memory/post_standard.md` (форматы/метрики),
`memory/format_playbook.md` (плейбук от Аналитика), `memory/post_lessons.md` (уроки) и сводка канала. Публикации нет — драфт автомат, публикует владелец. **Скаута не дёргает** —
читает готовую память напрямую.

## Петля обучения на правках
Владелец дорабатывает пост перед публикацией → присылает финал (`/feedback`) → Криейтор сравнивает
со своим драфтом, вычленяет устойчивые правила и пишет их в `memory/post_lessons.md` (грузится в
контекст к каждому посту). **Граница:** уроки — в слой памяти; личность (`SKILL.md`) Криейтор не
трогает (её правит Девелопер через гейт, PLAN §5.1). Устоявшийся урок Девелопер позже поднимет в SKILL.

## Железные правила (из SKILL)
- Не выдумывает цифры: чего нет — ставит `[ПРОВЕРИТЬ: …]`.
- Ссылки НЕ в текст поста (убивают охват) — только в брифе.
- Красные линии бренда; один драфт за раз; публикует владелец.

## Формат — под тему по данным (не «всегда флагман»)
Криейтор подбирает формат под тему по плейбуку Аналитика (`format_playbook.md`) и данным
(`top_posts`/`by_dimension`/`themes_overview`): флагман (2800–4096 знаков, медиа) для глубокой темы, лёгкий
философский (200–400) для ритма, образовательный («Словарь крипто») и др. Стандарты — **по
платформам** (сейчас Telegram; Threads и т.д. — отдельными файлами на Этапе 2).

## Стандарт через гейт (`/derive` → `/apply_standard`)
Стандарт выводится из практики, а не высечен: `/derive` собирает топ-посты + данные, выводит
редакторский стандарт и пишет **предложение** в `post_standard.proposed.md` (живой файл не трогает).
Владелец смотрит дифф → одобряет → `/apply_standard` копирует в `post_standard.md` с бэкапом в
`memory/.history/`. Деление ролей: «что заходит» (форматы/ER) ведёт Аналитик в `format_playbook.md`,
«как написать» — Криейтор. Расхождение выносится владельцу, не решается молча.

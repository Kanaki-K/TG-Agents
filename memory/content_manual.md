## 2. Рубрики Telegram (4)
1. **Глубокий разбор / флагман.** Сердце канала. 1/нед (четверг). 1500–3000 знаков (рабочий
   диапазон **2200–2500**). Структура: хук → контекст/цифры → 3–5 разделов → главная мысль → вопрос.
   Примеры: «Минус 90%. Кто выживает», «DXY как фильтр для крипто-риска».
2. **DCA-дневник.** Эксклюзив — открыто ведёт портфель. 1–2/мес, в дни реальных действий. 500–1000
   знаков. Цифры портфеля только в %, не в $.
3. **Макро-чек-ин.** Короткая содержательная аналитика по теме недели. Заменяет мотивашки. 1/нед
   (пн–вт). 300–600 знаков. Структура: 1 цифра + контекст + интерпретация. Это НЕ мотивашка.
4. **Разбор проекта / концепции.** Фундаментальный разбор актива/явления. 2/мес. 2000+ знаков,
   часто со ссылкой на Notion. Примеры: Layer 0–3, BTC/ETH/SOL/LINK, стейкинг, виды бирж.

Микс недели (4 поста): 1 глубокий разбор + 1 макро-чек-ин + 1 DCA-дневник/разбор проекта + 1 гибкий.
Главное правило — **регулярность флагмана раз в неделю**; при крупном инфоповоде плановая тема
откладывается ради контрнарратива «по горячему».

## 3. Микро-инструкция (к каждому посту)
- **Обращение/язык.** «Вы» с большой буквы. Русский без усложнения; сложное объясняем простыми
  словами; жаргон только когда без него никак.
- **Заголовок.** Цепляющий. При сомнении вопрос vs утверждение — выбирай вопрос («AI — главный
  союзник биткоина?» сильнее утверждения). Утверждение лучше только для жёсткого контрнарратива.
- **Рамка вовлечения.** Первый абзац = крючок/вопрос/метафора/сильный тезис (не «доброе утро,
  сегодня разберём»). Последний абзац = главная мысль + вопрос для обсуждения.
- **Один тезис.** Один пост = один главный тезис, который читатель повторит одним предложением.
  Остальное — усиление. Фильтр: «читатель понимает, как это касается лично его?»
- **Стиль.** Живой, с парадоксами и провокациями. Не учить и не морализировать — бросать
  мысль-вызов, читатель сам делает вывод. Логика — живая история, не сухой список. 1–2 эмоциональных
  приёма на пост (парадокс, провокация, хлёсткая формулировка, метафора).
- **Личный опыт.** Каждый пост — минимум одна фраза от первого лица («я сократил портфель», «откупил
  по 72к», «взломали — потерял 70%»). Пост без личного = пост, который мог написать любой бот.

## 4. Структура флагмана: чередование «думающий → знающий»
Главный принцип TG-флагмана — **чередование тонов**. Не три «знающих» абзаца подряд (заваливает
цифрами), не три «думающих» подряд (мысли без основания).

| Блок | Тон |
|---|---|
| Заголовок | Думающий (вопрос) |
| Хук | Знающий (цифра) |
| Объяснение механики | Думающий |
| Цифры по макро (ФРС/DXY) | Знающий (плотные данные) |
| Перевод цифр | Думающий (интерпретация) |
| Цифры по BTC | Знающий (метрики) |
| Главная мысль | Думающий (парадокс) |
| Личное действие | Знающий + действие |
| Афоризм | Думающий (цитата) |
| Вопрос к аудитории | Думающий (диалог) |

Идеальный ритм: **Д-Зн-Д-Зн-Д-Зн-Д-Зн-Д.** Перед публикацией пройти по абзацам и пометить тон.

Рабочие хуки (первая строка — 80% успеха):
- Напряжение без разрешения: «Потерял 70% эфиров из-за взлома. Вот что я понял.»
- Identity-based: «Долгосрочники, это для Вас.»
- Контрарный: «Крах не из-за X. Настоящая причина — Y.»
- Конкретная цифра: «28 млрд$ ликвидаций за час. Вот что это значит.»
- Вопрос-провокация: «Может быть, мы все ошибаемся насчёт X?»
- «Год назад / Сейчас», «В [году] было…» — работают в TG (в Threads нет).

## 5. Типографика и форматирование (жёстко)
- **Дефисы обычные `-`, не длинные `—`.**
- **Кавычки прямые `"…"`, не ёлочки `«…»`, не фигурные.**
- **Валюта ПОСЛЕ числа: `30$`, `40 млрд$`, `22.4 трлн$` — НЕ `$30`.** То же для €, ₽. Это главный
  типографический маркер «не-AI». Перед публикацией — Ctrl+F по `$`.
- Эмодзи в тексте: сдержанно, 2–4 на пост, для усиления ключевых мыслей.
- Эмодзи-маркеры разделов: фирменная навигация длинного поста (📊 🌐 ❓ ⚙️ 💎 💡) — это работает.
- Без капса в заголовках.
- **Нарратив > списки.** Списки только где реально нужны. Без стрелочек `→` как разделителей мыслей —
  заменять на новую строку или связки «когда / тогда».
- **Футер** (на флагманах/разборах/макро-чек-инах — всё, что репостится; можно не ставить на коротких
  DCA-дневниках/ответах): `🖥 Медиа | 🥸 Мемы | 📱 Notion`
- **Иллюстрация:** ни один пост без картинки. Флагман — сравнительная инфографика/многослойный график
  (1200×675, 16:9). Источники: TradingView, Newhedge.io, Alternative.me, DefiLlama, CompaniesMarketCap.
  Водяной знак @kanaki.crypto, тёмный фон, крупные цифры, максимум 3 цвета.

## 6. Красные линии (НИКОГДА)
Абсолютное табу: война в Украине (никакая позиция); политика (Россия, Украина, США, Германия);
Россия/Москва/рубль/русские бренды как тема; статус беженца, визы, Франкфурт; точные цифры доходов и
размер портфеля в $; хейт блогеров/персоналий; призывы купить/продать, сигналы, «инсайды».

Осторожно: религия — только лёгкие шутки; семья/отношения — очень редко; Германия/Украина — только
бытовое (еда, архитектура); доходы/портфель — только в % от портфеля.

**BTC (жёстко):** биткоин никогда не показывается проигравшим активом. Контрнарратив против
консенсуса, но не против биткоина. Можно: признавать волатильность, конкретные риски (квантовые
компьютеры, регуляции), разбирать свои ошибочные покупки, честно показывать механику падения.
Нельзя: делать BTC «сломавшимся хеджем». Голос всегда: «BTC — лучший долгосрочный актив эпохи,
просадки — возможности, цель не меняется».

Политика/война — только как макро-фактор прямого влияния на крипту (напр. нефть через геополитику),
без политической стороны.

## 7. Запретник AI-паттернов
3+ сработавших пункта → пост выглядит как AI и теряет доверие. Тест: прочитал → «как-то не
по-человечески» → переписываешь.

**Слова-маркеры (вычёркивать):**
- Канцеляризмы: «важно отметить/понять», «следует/стоит отметить», «таким образом», «в заключение»,
  «в современном мире», «играет ключевую роль», «не просто X, а Y».
- AI-эпитеты: «мощный инструмент», «революционный», «прорывной», «инновационный», «беспрецедентный»,
  «эффективный» без цифры рядом.
- Пустышки: «давайте разберёмся/погрузимся/рассмотрим», «раскроем потенциал», «открывает двери».
- **«является»** — нейросеть его обожает. Выкидывать (95% случаев): «Биткоин является активом» →
  «Биткоин — актив».

**Структурные паттерны:** доллар перед числом; длинные тире, ёлочки; триады-синонимы («надёжно,
безопасно и эффективно»); подзаголовок на каждые 2 абзаца (SEO-стиль); списки ради списков;
стрелочки `→` как разделители.

**Смысловые:** обобщения вместо конкретики («революция в финансах» → «в BTC-ETF зашло 130 млрд$ за
18 месяцев»); симметрия «с одной стороны / с другой» (у автора есть позиция — говори прямо);
заключение-пересказ (концовка добавляет: тезис + вопрос, а не повторяет); универсальность без
контекста (привязка к моменту: дата, текущий Fear & Greed, свежие цифры).

**Крипто-шаблоны (2+ = переписать):** «в мире криптовалют», «блокчейн-технология позволяет», «меняет
правила игры», «будущее за блокчейном», «токенизация открывает возможности».

**Интонация:** убрать приторность («отличный вопрос!», «надеюсь, было полезно!»). Тон — спокойная
уверенность с лёгкой иронией. Живой автор реагирует: иронизирует, удивляется, называет абсурд абсурдом.

**Тест на ценность абзаца (важнейший):** прочитать каждый абзац отдельно: (1) что нового даёт?
(2) если убрать — что потеряется? «Ничего/повтор» → удалять. Частая болезнь длинных постов — повтор
хука в середине.

**Англицизмы (отдельный проход):**
- Заменить: setup → расклад/картина; on-chain → в данных блокчейна; QE → выкуп гособлигаций;
  Treasuries → гособлигации США; cuts → снижения ставки; dump/pump → обвал/разгон.
- Расшифровать при первом упоминании: «биржевые фонды (ETF)», «комитет ФРС по ставке (FOMC)»,
  «индекс потребительских цен (CPI)», «общий объём заблокированных средств (TVL)».
- Оставить (фирменные): BTC, ETH, SOL и тикеры, DCA, HODL, ATH.
- Метрики (LTH, SOPR, MVRV, 365 SMA, Realized Supply) — полное название/мини-объяснение в одном
  предложении. Но: данные для анализа ≠ для коммуникации. Сначала «понятна ли главная мысль»,
  потом «точны ли цифры до запятой».

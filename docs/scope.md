# 🔭 «Под прицелом» — короткий аналитический пост (ветка scope)

Документация **ветки scope** для инженера: что это, зачем отдельно от флагмана, какие файлы, как
течёт, как запускать, как тюнить и что НЕ трогать.
Карта всего проекта — в [ARCHITECTURE.md](ARCHITECTURE.md). Стратегия и «почему» формата — в
[PLAN.md](PLAN.md) (§7 Backlog, рубрика). **Правила самой рубрики** (длина, приёмы подачи, красные
линии) — в [../memory/scope_manual.md](../memory/scope_manual.md): это «мануал», аналог
`content_manual.md` для флагмана.

> Решение 24.06.2026: scope вынесен в самостоятельную ветку, а не команду внутри флагман-Криейтера.

---

## 1. Что это и зачем ОТДЕЛЬНО

🔭 «Под прицелом» — короткий (~600–900 знаков тела + футер) аналитический пост: БЫСТРАЯ реакция на
важный внешний повод через линзу канона, **по гейту важности Скаута** (не по расписанию). ДЛИНА/плотность
— как тугие a01k (§8.2, вдвое короче флагмана); ЕДЖ/угол — по постам владельца (эталоны KANAKI §8.1, берётся
приём, не длина). Подача безличная, без флагман-фурнитуры-ярлыков (детали — в `scope_manual.md`).

Почему не команда внутри Криейтера, а отдельная ветка:
1. **Дёшево** — НЕ грузим в контекст флагман-«библию» (`content_manual.md`), блок обложки и плейбук:
   короткому это не нужно. Меньше токенов → дешевле прогон.
2. **Гибко** — свои правила (`scope_manual.md`) и своя модель настраиваются независимо от флагмана.
3. **Чисто** — флагман и scope не мешают друг другу: правка одного не задевает другой.

## 2. Файлы ветки

| Файл | Роль |
|---|---|
| `memory/scope_manual.md` | **Правила рубрики** (мануал): длина, приёмы подачи, анти-фурнитура, красные линии, эталоны. Тюнишь scope — правишь ЗДЕСЬ. Писатель: владелец. |
| `memory/scope_lessons.md` ★ | **Уроки из правок владельца** к 🔭-постам (петля обучения) — ОТДЕЛЬНО от флагман-`post_lessons.md`. Пишет инструмент `record_scope_lesson` (через `/scope_feedback`), грузится в `_system()`. Один факт — один дом. |
| `core/scope_writer.py` | **Ветка/код**: лёгкий `_system()` (персона + scope_manual + brand + уроки, БЕЗ флагман-мануала), `TOOLS` (руки Криейтера минус `make_image`), `SCOPE_MODEL`, `write()` = генерация + встроенный 2FA. |
| `core/creator_tools.py` `_lint` | Enforcement формата: длина >1100 ⛔ / <350 ⚠, фурнитура (💡/💭/«Вопрос к Вам»), ярлыки-разделы («Риск честно:» и пр.), срез `<cite>` веб-поиска. Общий линтер, ветка `kind='scope'`. |
| `core/content_plan.py` | Слот короткого (Пн/Ср/Пт) для отложки. |
| `data/creator_last_kind.txt` ★ | Формат последнего драфта → `_publish_now` ставит scope ТЕКСТОМ (обложку даёт только флагману). |
| `memory/post_standard.md` (Ф5) | Только **указатель** на эту ветку — флагман-Криейтер scope не пишет. |

Переиспользует (не дублирует): персону `agents/creator/SKILL.md`, «руки» `creator_tools` (save_draft/
read_brief/recent_posts/publish), `verify.py` (2FA), `agents/scout` (повод + флаг 🔭 в брифе).

## 3. Как запустить

```
python run_pipeline.py --scope     # вся цепь: Скаут (по гейту свежести) → scope_writer → отложка ТЕКСТОМ
```
Без флага `run_pipeline.py` — флагман (не затрагивается). В боте Криейтера:
- `/scope` — команда-**действие**: только ГЕНЕРИТ драфт (без Скаута/анти-повтора), дальше владелец `/schedule`.
- `/run_scope` — **ПОЛНЫЙ цикл из бота** (= `run_pipeline.py --scope`): свежесть→Скаут→анти-повтор→scope+2FA→
  отложка. Это `run_cycle(scope=True)` через ленивый импорт. `/run` — то же для флагмана. ⚠️ Тратит кредиты,
  сам ставит в «Отложенные», идёт минуты. Анти-повтор есть ТОЛЬКО в этом полном цикле, не в голом `/scope`.

**Петля обучения** (закрыта 26.06.2026): отредактировал короткий пост перед публикацией — пришли свой
финал командой `/scope_feedback <текст>` (в ТОМ ЖЕ сообщении). Ветка сравнит финал ↔ свой драфт и запишет
устойчивые уроки в `memory/scope_lessons.md` через `record_scope_lesson` (анти-дубль страж общий). Это
ОТДЕЛЬНО от флагман-`/feedback` (тот пишет в `post_lessons.md` на полном контексте) — короткий учится сам,
не мешая флагману. Урок-инструмент даётся модели ТОЛЬКО в режиме фидбэка, не при генерации.

> ⚠️ Качество scope требует боевой модели: запускай на `/main` (→ Sonnet). На `/test` (Haiku) формат
> плывёт — это потолок модели, не промпта.

## 4. Поток (через ту же ФАЙЛЫ-шину, что и флагман)

```
СКАУТ /scan → бриф с флагом формата 🔭 (memory/briefs/)            ★ШИНА★
      ▼
scope_writer.write():
  лёгкий _system() (scope_manual + brand + уроки, БЕЗ content_manual) на SCOPE_MODEL (Sonnet; /test→Haiku)
  → read_brief('latest'), пишет короткий по scope_manual
  → save_draft(kind='scope') → _lint (длина/фурнитура/срез <cite>)  ★ШИНА★ (+ data/creator_last_kind.txt)
  → ВСТРОЕННЫЙ 2FA: verify.verify_post → есть замечания → правит сам
      ▼
ПУБЛИКАТОР /schedule (или пайплайн): _publish_now видит kind=scope → отложка ТЕКСТОМ (без обложки),
  слот content_plan (Пн/Ср/Пт) → telegram_publish → ЛС владельцу
```

В пайплайне 2FA флагмана для scope **не дублируется** — он уже прошёл внутри ветки.

## 5. Чем отличается от флагмана

| | Флагман (Ф1) | 🔭 Под прицелом |
|---|---|---|
| Правила | `content_manual.md` | `scope_manual.md` |
| Код | `creator_bot` / `_run_creator` | `core/scope_writer.py` |
| Контекст `_system` | мануал + обложка + плейбук + стандарт + уроки | персона + scope_manual + brand + уроки (лёгкий) |
| Модель | Opus | Sonnet (`SCOPE_MODEL`) |
| Обложка | `make_image` (обязательна) | нет (инструмент не подключён) |
| Длина | 2800–4096, 15–30 цифр | ~600–900 тела (+футер), 1–3 цифры |
| Структура | ДАННЫЕ→МЕХАНИКА→МОРАЛЬ, 💡/💭, футер | приёмы подачи без фурнитуры (см. scope_manual §3–4) |
| Слот | Вт/Чт | Пн/Ср/Пт |
| Обучение | `/feedback` → `post_lessons.md` | `/scope_feedback` → `scope_lessons.md` |

## 6. Как тюнить

- **Правила/подача/длина** → `memory/scope_manual.md` (одно место, флагман не затронешь).
- **Модель** → `SCOPE_MODEL` в `core/scope_writer.py` (через `runmode`: `/test` всё равно даёт Haiku).
- **Жёсткие пороги** (длина >1100 / <350, фурнитура, ярлыки-разделы, срез `<cite>`) → ветка `kind='scope'` в `creator_tools._lint`.

---

## 7. ⚠️ Что НЕ трогать не подумав

1. **Не грузить `content_manual.md` в scope-контекст** — это вернёт флагман-замашки и цену. Лёгкий
   `_system()` — суть ветки.
2. **`data/creator_last_kind.txt`** — по нему `_publish_now` отличает scope (текст) от флагмана (фото).
   Сломаешь запись `kind` в `save_draft` → короткий уедет с чужой обложкой из `creator_last_cover.txt`.
3. **Срез `<cite>` в `_lint`** — разметка веб-поиска протекает в текст у слабых моделей; не убирать.
4. **Гейт важности** — scope только по значимому поводу (флаг 🔭 Скаута). Конвейер «N в неделю по
   расписанию» = тухло (PLAN/решение 24.06). Частота — следствие гейта.
5. **Не сливать `scope_lessons.md` и `post_lessons.md`** — у короткого и флагмана РАЗНЫЕ уроки
   (длина/подача/масштаб), смешение вернёт флагман-замашки в короткий. `_system()` scope грузит ТОЛЬКО
   `scope_lessons.md`; урок-инструмент `record_scope_lesson` даётся лишь в `write_feedback`, не при генерации.

# Бэклог апгрейдов — Криейтор и контент-система

> Слабые места и улучшения, выявленные в ревью 17.06.2026. Ранжировано по важности. К этому
> возвращаемся после первого живого прогона создания поста. Статусы обновляем по ходу.

## 🔴 Критичное (надёжность)

### 1. Прогнать вживую (СНАЧАЛА)
Ничего не выполнялось на реальных ботах с API. Классификация, флагман по мануалу, петля `/feedback` —
непроверены. Первый прогон вскроет косяки. **Действие владельца:** Аналитик `/classify`+`/playbook` →
Скаут `/scan` → Криейтор `/post` → правка → `/feedback`. Прислать результат → чинить прицельно.

### 2. Гейты качества — в КОД, не в промпт
Типографика/запретник/чек-лист/свежесть живут в промпте → при компакции контекста молча отваливаются
(урок OpenClaw). **Сделано до прогона:** код-линтер в `save_draft` (авто-чинит тире/кавычки, варнит
валюту-перед-числом, размер флагмана, футер, «является»; возвращает вычищенную версию). **Осталось:**
расширить (анти-AI слова-маркеры списком, проверка «нет ссылок в теле», структурные проверки).

### 3. Кэширование промпта + дедуп правил
Контекст Криейтора большой (мануал+brand+standard+playbook+lessons+summary) на каждый вызов Opus —
дорого и «разжижает» соблюдение правил. **Действие:** prompt caching (cache_control на статичную
память, −90% на кэше, PLAN §8) + убрать дубль красных линий (brand vs мануал). Не делал до прогона:
требует теста, ошибка ломает все вызовы.

## 🟡 Важное

### 4. Эвалы — страховка петли обучения
`/feedback` копит уроки, `/derive` меняет стандарт — без регрессионного сигнала это дрейф. **Действие:**
11 живых флагманов как «золотой эталон» + скоринг нового драфта по чек-листу (зачаток `tests/`, Этап 7).

### 5. Стык Скаут → Криейтор (импеданс)
Скаут отдаёт ресёрч-дамп (5 направлений + цифры); флагман строится ОТ антитезы + история-как-аргумент.
Криейтор скатывается в «дашборд». **Действие:** Скауту отдавать под тему ещё кандидат-АНТИТЕЗУ +
исторический параллель, не только свежие цифры. Самый недооценённый рычаг качества флагмана.

### 6. Данные — один снимок
Аналитика на экспорте ~11.06 (#437). Классификация/плейбук/свежесть на нём. **Действие:** дисциплина
`/update` (нужна telethon-сессия владельца); решения свежи настолько, насколько свеж экспорт.

### 7. Разметка форматов не проверена владельцем
19 флагманов размечены эвристикой (медиа+450 слов); плейбук/стандарт зависят от ярлыков. **Действие:**
спот-чек `by_format('флагман')` → правки `set_format`; при желании LLM-доразметка спорных средних.

## 🟢 На потом

### 8. Анти-самоповтор без памяти
Сверка вживую каждый раз; нет леджера приёмов → не поймать «временной якорь 3 из 5». **Действие:**
структурное поле приёмов (хук-тип/антитеза/вопрос) на пост для острой сверки.

### 9. Хрупкий порядок запуска
Криейтор зависит от того, что `/classify`+`/playbook` прогнаны ДО `/post`. **Действие:** оркестратор
(Этап 5) либо предупреждение Криейтора о пустом/старом плейбуке.

### 10. «Рук» нет (постинг/картинки/планировщик)
Всё кончается драфтом в чат. Петля «опубликовал → измерил → учёл» не крутится, пока не публикуешь →
`/update`. **Действие:** автопостинг + генерация иллюстраций — отдельный этап (осознанно за гейтом «ок»).

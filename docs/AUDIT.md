## Как обновлять этот файл

1. **Переаудит** (раз в N недель или после крупных изменений): прогнать 4 оси параллельными
   read-only агентами — архитектура, безопасность, качество кода, инженерная зрелость. Промпты-скелеты
   и охват — см. «Методология» ниже. Каждая ось → балл 1–10 + находки с `file:line`.
2. **Обновить баллы** в таблице «Скоринг» (добавить новую строку с датой, старые НЕ стирать — видно динамику).
3. **Статусы фиксов:** `☐` открыто · `◐` в работе · `☑` сделано (дата + хеш коммита) · `🗑` отменено (почему).
   Новые находки добавлять с новым ID (не переиспользовать освободившиеся).
4. **Changelog** внизу — одна строка на изменение аудита.
5. **Не регрессировать сильные стороны** (раздел ниже) — это то, что уже сделано правильно.

## Методология (чтобы переаудит был воспроизводим)
4 независимые оси, каждую ведёт отдельный read-only агент (general-purpose), синтез — у ведущего:
- **Архитектура:** слои (движок/руки/агенты/память), «мозги vs руки», DRY, coupling, расширяемость,
  согласованность ARCHITECTURE.md с кодом. Читать: `core/{agent_runtime,llm,config,memory,analytics}.py`,
  все `*_bot.py`/`*_tools.py`, `workshop.py`, `agents/*/`.
- **Безопасность:** секреты (.env/config), `.gitignore`, коннекторы (twikit-куки, telethon-сессии),
  subprocess/eval, авторизация ботов, гейт девелопера. grep на хардкод-ключи.
- **Качество кода:** обработка ошибок (Claude API, сеть, парсинг), async-корректность (блокировки loop),
  дублирование, робастность JSON/CSV/MD, логирование, type hints.
- **Инженерная зрелость:** тесты, CI, lock-файл, Docker/процесс-менеджмент, бэкапы, документация, онбординг.

## Сильные стороны (НЕ регрессировать)

- **«Мозги vs руки» реально соблюдён.** `agent_runtime.run()` параметризован (`tools_schema`/`dispatch`/
  `system_builder`) — движок не знает агентов, агенты импортируют движок. Инверсия зависимостей.
- **DRY-рантайм:** один движок на 5 агентов, `*_bot.py` — тонкая декларативная обвязка (40–175 строк).
- **Структурные гейты кодом, не промптом:** `workshop.apply()` промоутит только существующий `.proposed`
  (+ бэкап в `.history/` + защита от path-traversal); `creator._apply_standard` — так же.
- **Грабли Anthropic API учтены:** починка обрыва `tool_use/tool_result` (`_trim_history`), пустой
  `tool_result`→заглушка, `pause_turn`, `busy`-замок от дублей, `to_thread` для синхронного SDK.
- **Гигиена секретов:** `.gitignore` исчерпывающий (`.env*`, `*.session`, `data/`, owner-файлы); ноль
  хардкод-ключей; `yaml.safe_load`; subprocess без `shell=True`/ввода пользователя.
- **Детерминированные линтеры** (`creator_tools._lint`): типографика/размер/стаккато/жирный кодом, счёт
  длины в UTF-16 под лимит Telegram.
- **Документация объясняет решения** (ARCHITECTURE.md: почему код агентов плоско в `core/`).
